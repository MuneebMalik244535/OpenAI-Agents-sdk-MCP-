# Evaluation in OpenAI Agents SDK

## Overview

Evaluation (Evals) is the process of testing and measuring an AI agent's
performance.

It helps verify:

-   Accuracy
-   Reliability
-   Quality of responses
-   Correct tool usage
-   Workflow behavior

Evaluations are important because AI systems can produce different
outputs for the same input, so traditional software testing is not
enough.

------------------------------------------------------------------------

# Why Evaluation is Needed

Evals help developers:

-   Detect agent failures
-   Compare different models or prompts
-   Measure improvements
-   Prevent regressions after changes
-   Build reliable AI applications

------------------------------------------------------------------------

# Types of Evaluation

## 1. Output Evaluation

Checks the final response generated by the agent.

Measures:

-   Correctness
-   Format
-   Style
-   Relevance
-   Safety

------------------------------------------------------------------------

## 2. Workflow / Trace Evaluation

Evaluates the complete agent execution flow.

It checks:

-   Agent decisions
-   Tool calls
-   Tool arguments
-   Intermediate steps
-   Final result

Trace grading helps identify where an agent succeeded or failed inside
the workflow.

------------------------------------------------------------------------

## 3. Tool Evaluation

Checks whether tools are:

-   Selected correctly
-   Called with correct parameters
-   Producing expected results

------------------------------------------------------------------------

# Evaluation Process

A typical evaluation workflow:

    Define expected behavior
            |
            ↓
    Create test cases
            |
            ↓
    Run agent evaluations
            |
            ↓
    Analyze results
            |
            ↓
    Improve agent

------------------------------------------------------------------------

# Datasets

Datasets are collections of test examples used to evaluate agents.

A dataset contains:

-   User inputs
-   Expected outputs
-   Evaluation criteria

They help quickly test prompts and agent behavior.

------------------------------------------------------------------------

# Graders

Graders evaluate agent outputs automatically.

They compare:

-   Expected results
-   Agent-generated results

Graders can provide:

-   Pass/fail results
-   Scores
-   Labels

------------------------------------------------------------------------

# Types of Graders

## String Check Grader

Checks exact text matches.

## Text Similarity Grader

Measures similarity between responses.

## Score Model Grader

Uses a model to assign quality scores.

## Python Code Grader

Uses custom code logic for evaluation.

------------------------------------------------------------------------

# Agent Evaluation Best Practices

## Create Clear Criteria

Define what a successful response means.

## Test Multiple Cases

Include:

-   Normal inputs
-   Edge cases
-   Failure scenarios

## Track Changes

Compare agent versions after:

-   Prompt changes
-   Model changes
-   Tool changes

------------------------------------------------------------------------

# Evaluation vs Tracing

## Evaluation

Answers:

"Is my agent performing well?"

Used for:

-   Quality measurement
-   Comparing versions
-   Regression testing

## Tracing

Answers:

"What happened during this agent run?"

Used for:

-   Debugging
-   Understanding workflow
-   Finding failures
